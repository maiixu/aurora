# Aurora — Development Guide

## Directory structure

```
src/
  main/                   # Electron main process (Node.js)
    pasters/              # Per-app paste strategies
      index.ts            #   getPaster(appName) registry — add new apps here
      types.ts            #   Paster interface
      ax.ts               #   Default: click Edit > Paste via UI scripting
      iterm2.ts           #   iTerm2: direct session write
      messages.ts         #   Messages.app: activate + click Paste menu
    index.ts              # App entry: whenReady, permissions, tray, hotkey, backend
    state-machine.ts      # FSM: IDLE → LISTENING → PROCESSING → TRANSCRIBING → READY/CANCELLED
    hotkey.ts             # Electron globalShortcut (Hyper+F13 via Karabiner, toggle mode)
    hud-window.ts         # HUD BrowserWindow (panel, non-activating); resizes dynamically
    ipc-handlers.ts       # IPC bridge: audio → transcribe → paste; HUD resize
    tray.ts               # Menu bar tray + backend/model switcher + TrayAnimator wiring
    tray-animator.ts      # Per-frame dot animation for each FSM state
    transcriber.ts        # Thin wrapper → transcribe-ec2.ts
    transcribe-ec2.ts     # POST audio to whisper-server; handles SSE (EC2) and JSON (local)
    whisper-tunnel.ts     # Spawn SSH tunnel to EC2; probe port for existing tunnel
    whisper-local.ts      # Manage local whisper-server process; PID file; model selection
    aurora-config.ts      # Read/write ~/.aurora/config.json (backend mode, localModel)
    permissions.ts        # Mic + Accessibility permission checks
  preload/
    hud-preload.ts        # contextBridge: onStateChange, onToken, resizePanel, ready, sendAudio
  renderer/hud/
    main.ts               # Canvas setup, state/audio/token wiring
    animator.ts           # Canvas dot animation + TRANSCRIBING text panel; dynamic resize
    recorder.ts           # getUserMedia → WAV encoder
    volume-meter.ts       # Web Audio RMS → animator.setVolume
    styles.css
  shared/
    types.ts              # AppState enum, IPC channel constants
    constants.ts          # HUD sizing, animation timings
assets/
  tray-icon.png / @2x     # Generated by gen-dot-icon.swift (do not edit manually)
```

## Git workflow

### Daily

```bash
git pull --rebase          # always rebase, never merge-pull
git push                   # push after every logical commit
```

### Feature work

```bash
git checkout -b feat/<name>   # new feature
git checkout -b fix/<name>    # bug fix
# ... work ...
git checkout main
git merge --no-ff feat/<name>
git push
git branch -d feat/<name>
```

- `main` is always shippable — never commit broken code directly
- No long-lived branches; merge when the feature is complete and tested
- Delete branches after merge

### Commits

- One logical change per commit — don't bundle unrelated changes
- Format: `type: short description`
  - `feat:` new capability
  - `fix:` bug fix
  - `refactor:` restructure without behaviour change
  - `chore:` deps, config, tooling
- Keep the message under 72 chars; use body for context if needed

## Code hygiene

### Before closing a task, check for dead code

```bash
# Find files with no imports anywhere in src/
for f in $(find src -name "*.ts"); do
  name=$(basename $f .ts)
  grep -r "$name" src/ --include="*.ts" -l | grep -v "$f" > /dev/null || echo "UNUSED? $f"
done
```

- Delete files that have zero imports and are not entry points
- Remove constants/types that nothing references
- When a module is replaced (e.g. rewriting animator.ts inline), delete the old files immediately — don't leave them as "maybe useful later"

### Adding a new app paster

1. Create `src/main/pasters/<ProcessName>.ts` implementing `Paster`
2. Register it in `src/main/pasters/index.ts`
3. App name = the string System Events returns for that process (usually matches the .app name without `.app`)

```typescript
import { execSync } from 'child_process'
import type { Paster } from './types'

export const myAppPaster: Paster = {
  paste(_text: string) {
    execSync(`osascript -e '
      tell application "MyApp" to activate
      delay 0.15
      tell application "System Events"
        tell process "MyApp"
          click menu item "Paste" of menu 1 of menu bar item "Edit" of menu bar 1
        end tell
      end tell
    '`)
  },
}
```

### HUD / tray

- Tray icon is pixel-rendered in `TrayAnimator` — no external asset needed, no Sketch/Figma
- HUD dot size is computed at runtime from screen height (`sh / 40`, clamped 22–48px) — never hardcode px
- To regenerate `assets/tray-icon.png` run `/tmp/gen-dot-icon.swift` via swiftc (source lives in git history)

## Environment variables

| Variable | Default | Purpose |
|---|---|---|
| `AURORA_SSH_HOST` | `mac-ec2` | SSH config alias for EC2 whisper server |
| `AURORA_WHISPER_PORT` | `8080` | Remote whisper-server port on EC2 |
| `AURORA_DEVTOOLS` | unset | Set to any value to enable Chrome DevTools on port 9222 (dev only) |

## Runtime config (`~/.aurora/config.json`)

| Key | Default | Values |
|---|---|---|
| `backend` | `"local"` | `"local"` \| `"ec2"` \| `"auto"` |
| `localModel` | `"large-v3"` | stem of any `.bin` in `~/.aurora/models/` |

`auto` mode: tries EC2 first, falls back to local after 6s if EC2 unavailable.

## macOS permissions

| Permission | Why |
|---|---|
| Microphone | Record audio |
| Accessibility | Paste via UI scripting (Aurora itself — no helper binary needed) |

Input Monitoring is NOT required — hotkey uses Electron `globalShortcut` (Carbon `RegisterEventHotKey`), not uiohook.

## Development learnings

Accumulated troubleshooting notes, paste strategy decisions, and AppleScript gotchas are in **[PROGRESS.md](PROGRESS.md)**. Read it before debugging paste issues or adding a new app paster. Update it whenever a non-obvious problem is solved.

## Security invariants (don't break these)

- **DevTools** are gated on `AURORA_DEVTOOLS` env var — never re-enable by default
- **`/tmp` logging** must not contain transcribed text or app names — world-readable
- **EC2 port 8080** must not be exposed in Security Group — tunnel only
- **SSH key** for EC2 must be a dedicated `aurora_ec2` key, not shared/course keys
- **Paste path**: all osascript runs from Aurora's main process — child processes don't inherit Accessibility trust

## Key design decisions (don't re-litigate without good reason)

- **Toggle hotkey** (press once = start, press again = stop) — simpler than hold, no Karabiner hold-threshold needed
- **globalShortcut not uiohook** — uiohook's CGEventTap (active tap) intercepts ALL keyboard events system-wide, causing Escape and other keys to lag/drop. Electron `globalShortcut` uses Carbon `RegisterEventHotKey` which only fires for the registered combo. Karabiner maps Dictation key → `Cmd+Ctrl+Alt+Shift+F13` (Hyper+F13).
- **No paste-helper binary** — Aurora's own Accessibility permission is enough; child processes don't inherit it
- **Dual whisper backend** — local whisper.cpp (default) for offline/fast; EC2 via SSH tunnel for remote. Both hit `localhost:18080/inference`. `transcribe-ec2.ts` detects `Content-Type` to handle SSE (EC2 custom build) vs JSON (standard whisper.cpp).
- **HUD is a non-activating panel** — `type: 'panel'` means it never steals focus from the target app
- **Paster registry pattern** — each app gets its own paste strategy; default is click Edit > Paste via osascript
- **Bundle ID `com.maixu.aurora`** — do NOT revert to `com.aurora.app`. macOS Sequoia's Gatekeeper permanently caches a broken signature for that ID after `codesign --force --deep` was run, causing NSStatusItem to be silently suppressed.
- **update-app deploy pattern** — replace only `.asar` + `.asar.unpacked`, never `cp -r` the whole `.app`. Overwriting the app bundle with a freshly electron-builder-packaged bundle breaks the ad-hoc signature (gets `flags=0x20002(linker-signed)` instead of `0x2(adhoc)`), causing menu bar icon to vanish.
